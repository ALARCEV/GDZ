# Docs Source

This directory stores the raw product source materials copied into the repository for Codex-driven docs work.

## Rules
- Treat these files as upstream inputs, not normalized working docs.
- Prefer updating files in `docs/` when clarifying structure or wording.
- Replace files in this directory only when a new upstream source package is provided.
- If `docs/` diverges from these sources, document the reason in the relevant task or commit.
