# Школьный ассистент — MVP продукта и воронки

## Что это
Простой mobile-first сервис в формате чата для родителей и школьников. Главный сценарий: быстро объяснить тему, помочь с домашним заданием, разобрать фото страницы или голосовой вопрос и дать понятный, спокойный, человеческий ответ.

## Главная бизнес-логика
Продукт запускается как **utility-first acquisition tool**.

Сначала пользователь получает реальную пользу бесплатно. Затем сервис мягко предлагает сохранить чат, открыть историю, продолжить после лимита или получить дополнительные возможности. В этот момент происходит сбор контактов, прежде всего контактов родителя.

Модель:
- 1–3 месяца полностью бесплатно за счет основателя
- цель этапа: собрать базу, привычку использования и доверие
- затем переход к оплате по себестоимости с запасом 5–10%
- позже подключаются платные надстройки и pro-функции

## Основной пользователь
**Primary user:** родитель  
**Secondary user:** ученик

MVP-связка:
- один родитель
- один ребенок
- один выбранный класс
- все школьные предметы
- фокус только на школьной программе

## MVP-функциональность
- чат с минимальным интерфейсом
- guest mode без регистрации
- текстовые вопросы
- фото учебника или задания
- голосовые сообщения
- объяснение простыми словами
- пошаговое решение
- готовый ответ с разбором
- несколько способов решения
- отдельный режим ответа: как родителю помочь ребенку самому
- сохранение истории
- группировка по классу, предмету, теме и дате
- фильтры по предмету, теме и периоду

## Воронка
### 1. Вход
Пользователь заходит без регистрации и сразу попадает в чат.

### 2. Первичная польза
Он получает 1–2 сильных ответа без capture.

### 3. Capture
Сбор данных происходит не до пользы, а после нее. Основные триггеры:
- сохранить чат
- открыть историю
- продолжить после бесплатного лимита
- получить дополнительные фото/голосовые возможности
- получить памятку для родителя

### 4. Registration flow
Для родителя:
- имя
- роль
- класс ребенка
- email
- телефон опционально или обязательно по эксперименту

Для ученика:
- роль
- возраст
- email

Опционально:
- вход одной кнопкой через Яндекс

### 5. Retention
Возврат строится через:
- сохраненную историю
- удобные фильтры
- продолжение прошлых диалогов
- мягкие рассылки родителям

## Согласия и данные
Принцип: сначала ценность, потом данные.

Собирать в MVP:
- роль
- имя или никнейм
- класс
- email
- телефон по эксперименту
- источник трафика
- согласие на обработку данных
- согласие на маркетинговые сообщения

Не собирать без необходимости:
- полное ФИО ребенка
- школу
- адрес
- лишние поля

## AI-логика
- отдельные prompt profiles по классам
- ограничение ответов школьной программой
- стиль объяснения: просто, дружелюбно, без занудства
- ответы должны подходить и родителю, и ребенку
- работа с фото идет через vision-обработку в рамках мультимодальной модели
- работа с голосом идет через speech-to-text и затем в LLM

## Важное архитектурное решение по математике
Для формул, степеней, логарифмов, уравнений и прочих точных математических объектов нельзя опираться на generative image-модель как на основной способ визуализации.

Правильный путь:
- превращать математическую запись в точную разметку
- рендерить ее в SVG или PNG
- показывать на белом фоне черным текстом
- рядом давать человеческое объяснение

Это дает точность и читабельность.

## Админка MVP
Нужен не тяжелый backoffice, а компактная operational admin panel.

Должно быть:
- dashboard
- лимиты по ответам, фото, голосу
- редактор prompt profiles
- просмотр базы пользователей
- статусы guest / lead / active / paid / dormant
- ручное начисление запросов
- блокировка abuse
- тексты баннеров и capture-окон
- подготовка к будущей плате по себестоимости

## Монетизация
### Этап 1
Полностью бесплатный тестовый период.

### Этап 2
Оплата по понятным пакетам:
- 10 дней
- 50 запросов
- 100 запросов

Важно:
пользователь покупает не токены, а понятную единицу доступа.

### Этап 3
Платные надстройки:
- экспорт в Obsidian
- учебные планы
- семейный кабинет
- аналитика прогресса

## Техническая форма
- mobile-first web app
- адаптация под desktop
- код и компоненты должны быть готовы к будущему app wrapper
- удобно интегрировать UI-блоки и секции с 21st.dev
- backend должен закрывать auth, chats, files, quotas, consent logs, usage tracking, admin и billing hooks

## Метрики
- сколько гостей получили value moment
- сколько сохранили чат
- сколько оставили контакт
- сколько вернулись на 2, 7 и 14 день
- какие предметы лучше удерживают
- какая средняя стоимость активного пользователя
- какая конверсия в оплату после free phase

## Бренд
- отдельный нейтральный бренд для родителей
- название на русском
- 2–3 слога
- легко произносится
- легко пишется
- аккуратно транслитерируется
- образ бренда: добрый, современный, понятный, не инфантильный
