- Школьный ассистент — MVP продукта и воронки
  - 1. Стратегическая рамка
    - Utility-first продукт: сначала польза, потом capture и монетизация
    - Главная цель: взять базу родителей через реальную учебную ценность
    - MVP-фокус: 1 родитель → 1 ребенок → 1 класс
    - Период запуска: 1–3 месяца бесплатно за счет основателя
    - Переход к оплате: по себестоимости + 5–10% запас на курс, комиссии и потери
    - Позиционирование: спокойный, понятный, полезный помощник, не занудный репетитор
  - 2. Основные пользователи
    - Родитель — primary user
      - Хочет быстро понять задачу и помочь ребенку без стресса
      - Хочет простой язык, пошаговый разбор, уверенность и экономию времени
      - Ценит удобство, честность, безопасность и ощущение поддержки
    - Ученик — secondary user
      - Пользуется сам или вместе с родителем
      - Нужны понятные примеры, аналогии, метафоры, friendly tone
      - Цель продукта: перевести ребенка к самостоятельным сессиям за ~14 дней
    - Роли в системе
      - Гость
      - Родитель
      - Ученик
      - Администратор
  - 3. Продуктовый MVP
    - Интерфейс: сверхминималистичный чат, как открытое окно
    - Платформа: mobile-first web, хорошо работает на ПК и телефоне
    - Архитектура UI: готовность к дальнейшему оборачиванию в Android / iOS
    - Входные форматы
      - Текст
      - Фото страницы / задания
      - Голосовые сообщения
    - Основные сценарии
      - Объясни тему простыми словами
      - Помоги решить задачу
      - Дай готовый ответ с разбором
      - Покажи несколько способов решения
      - Подскажи, как родителю помочь ребенку самому
      - Дай независимую консультацию по школьной программе / плану
    - Предметы MVP
      - Все актуальные школьные предметы
      - Фокус ограничен школьной программой
      - Продвинутые учебные планы и надстройки уходят в future paid layer
  - 4. Воронка MVP
    - Acquisition
      - Трафик идет на нейтральный бренд для родителей
      - Коммуникация на русском языке, простое и теплое позиционирование
      - Задача рекламы: показать моментальную полезность, а не сложную платформу
    - Вход без трения
      - Guest mode без регистрации
      - Пользователь сразу попадает в чат
      - Стартовые CTA: объяснить тему / разобрать фото / помочь с ДЗ
      - Можно получить первые 1–2 сильных ответа без capture
    - Момент ценности
      - Первый ответ должен быть простым, спокойным, полезным и визуально ясным
      - Родитель видит: сервис реально экономит время и снижает стресс
      - Ребенок видит: ассистент говорит понятнее учебника
    - Capture после ценности
      - Триггер 1: сохранить чат
      - Триггер 2: открыть историю
      - Триггер 3: продолжить после лимита бесплатных ответов
      - Триггер 4: открыть голосовой / фото-режим сверх базового лимита
      - Триггер 5: получить памятку для родителя
    - Registration flow
      - Минимальная регистрация после ценности
      - Поля для родителя: имя, роль, класс ребенка, email и/или телефон
      - Поля для ученика: роль, возраст, email
      - Опционально: вход одной кнопкой через Яндекс
      - Telegram login не делать базовой опорой в MVP
    - Retention loop
      - История сохраненных чатов
      - Возврат к предметам и темам
      - Продолжение прошлых диалогов
      - Напоминания и мягкие рассылки для базы родителей
    - Future monetization entry
      - После 1–3 месяцев бесплатного периода
      - Пакеты по понятным единицам: 10 дней / 50 запросов / 100 запросов
      - Про-надстройки: экспорт, учебные планы, семейный кабинет, аналитика
  - 5. Регистрация, контакты и согласия
    - Главный принцип: сначала польза, потом данные
    - Главный контакт для базы: родитель
    - Что собирать в MVP
      - role
      - parent_name or nickname
      - child_grade
      - email
      - phone optional or required by experiment
      - traffic_source
      - consent_personal_data
      - consent_marketing
    - Что не собирать без необходимости
      - Полное ФИО ребенка
      - Школу
      - Адрес
      - Лишние данные на всякий случай
    - Логика согласий
      - Отдельное согласие на обработку данных
      - Отдельное согласие на маркетинговые рассылки
      - Логирование времени, версии текста и источника согласия
  - 6. UX и IA
    - Главные экраны
      - Landing / entry
      - Chat
      - Capture modal / registration
      - History
      - Profile
      - Admin
    - Структура истории
      - История нужна с первого релиза
      - Группировка: класс → предмет → тема → чаты
      - Автоматическая тема на основе содержания диалога
      - Дата как обязательный атрибут
    - Фильтры истории
      - По дате / месяцу
      - По предмету
      - По теме
      - По роли пользователя
    - Поведение чата
      - Стартовые подсказки и quick actions
      - Мягкий empty state без перегруза
      - Сохранение контекста сессии
      - Честное сообщение о бесплатном тестовом периоде
    - Tone of voice
      - Спокойный
      - Понятный
      - Небюрократичный
      - Подходит и ребенку, и родителю
  - 7. AI-слой и логика ответов
    - Системные промпты
      - Отдельные prompt profiles по классам 1–6 / 1–7
      - Промпт ограничивает ответы школьной программой
      - Промпт упрощает, не сюсюкает и не уходит в академический туман
      - Промпт всегда ищет лучший способ объяснить
    - Режимы ответа
      - Объяснение простыми словами
      - Пошаговое решение
      - Готовый ответ + разбор
      - Несколько способов решения
      - Режим для родителя: как помочь ребенку самому
    - Работа с фото
      - Фото страницы, упражнения, задачи
      - Vision-модель считывает содержание без отдельной OCR-цепочки в MVP
      - Ассистент сначала понимает задачу, потом объясняет
    - Работа с голосом
      - Голос → распознавание → нормализация запроса
      - Вопрос голосом должен ощущаться таким же простым, как чат
    - Математика и формулы
      - Формулы и уравнения выводить не generative image-моделью
      - Использовать точный рендер из математической разметки в SVG/PNG
      - Белый холст, черный текст, максимальная читабельность
      - Рядом давать текстовое объяснение человеческим языком
    - Guardrails
      - Не подменять обучение голыми ответами без логики
      - Не уводить ребенка в неподходящий контент
      - Не усложнять объяснение
      - При сомнении задавать короткий уточняющий вопрос

      - Ответы в день
      - Ответы за период

      - Голосовой лимит
      - Глубина ответа
    - Промпты
      - Редактор prompt profiles

    - Пользователи и CRM
      - Просмотр базы
      - Сегменты: родители / ученики / активные / спящие

      - Ручное начисление запросов

    - Контент и сообщения
      - Баннеры и тексты о тестовом периоде
      - Тексты capture-окон
      - Сообщения о переходе к оплате
  - 10. Монетизация
    - Free phase
      - Полностью бесплатный период 1–3 месяца
      - Задача: база, доверие, первые привычки использования
    - Cost-based access
      - Не продавать токены как токены
      - Продавать понятные пакеты доступа
      - База расчета: фактическая себестоимость запросов
      - Наценка: +5–10% safety margin
    - Пакеты
      - 10 дней
      - 50 запросов
      - 100 запросов
      - Позже: семейный / расширенный / pro
    - Переход к про-функциям
      - Экспорт в Obsidian
      - Учебные планы
      - Отчеты по прогрессу
      - Семейный кабинет
  - 11. Аналитика и метрики
    - Сколько гостей дошло до первого value moment
    - Сколько сохранило чат
    - Сколько оставило контакт
    - Сколько вернулось на 2-й, 7-й, 14-й день
    - Какие предметы дают наилучший retention
    - Средняя стоимость одного активного пользователя
    - Средняя стоимость одной сохраненной сессии
    - Конверсия в платный пакет после free phase
  - 12. Техническая архитектура
    - Frontend: mobile-first web app
    - Component-first UI, удобный для вставки 21st.dev блоков
    - Backend: auth, chats, files, quotas, billing hooks, admin
    - Storage: users, chats, attachments, consents, usage
    - AI bridge: prompt orchestration → multimodal model → formatted response
    - Voice pipeline: upload → speech-to-text → LLM
    - Formula renderer: math notation → SVG/PNG
    - Future-ready for PWA and app wrapper
  - 13. Бренд и коммуникация
    - Нейтральный бренд для родителей, не личный бренд основателя
    - Название на русском, 2–3 слога, легко произносится и пишется
    - Транслитерация должна быть простой
    - Без сложных шипящих и спорных буквосочетаний
    - Образ бренда: добрый, понятный, современный, не инфантильный
